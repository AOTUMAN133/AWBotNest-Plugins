# 115历史扫描 (my115scan) 更新说明

## v0.10.0 → v0.10.12

### ✨ 新增功能
- **Emby 数据库直读** — 配置 `library.db` 路径后直接读 SQLite，毫秒级获取所有 TMDB ID（232,604个），不再依赖 API 分批拉取
- **缓存独立建立** — 新增「建立缓存」按钮，与扫描分离；配置界面显示缓存进度和状态
- **查重方式可选** — 支持「缓存(全量拉取)」和「实时查询(逐条)」两种模式
- **扫描进度日志** — 每 50 条显示一次进度

### 🔧 优化
- 超时自动重试 3 次（间隔递增 5s→10s→15s），不直接放弃
- 减少 API 调用，直接使用原始消息对象处理
- 翻页日志便于排查扫描不全问题

### 🐛 修复
- 排除类型 `matched` 变量未定义导致异常被吞、排除失效
- 完结正则不匹配 `(完` 格式
- `media_types` 字符串/列表类型不兼容导致全部跳过
- 转存后实时更新缓存，防止同批重复
- Emby 查询失败时跳过不转发，避免重复入库
- 表名大小写敏感问题（Emby 数据库 `MediaItems` vs `mediaitems`）
- 数据库 `ProviderIds` 列格式适配（`Tmdb=10092|Imdb=...`）
- 配置关闭重开后扫描状态自动恢复
